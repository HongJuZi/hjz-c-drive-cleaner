迁移策略

三种智能迁移策略

根据工具特性自动选择最优方案，确保迁移安全可靠

Junction 软链接

通过创建 NTFS 目录联接（Junction Point），将原始路径指向新位置。对软件透明，无需修改配置，适合不支持环境变量的工具。

适用于：IDE、浏览器、设计软件

环境变量迁移

通过设置系统环境变量，引导工具将缓存写入新路径。最安全的方式，软件原生支持，无副作用，可随时清除变量恢复。

适用于：包管理器、编译器、CLI工具

虚拟磁盘专属

针对 Docker、WSL2、VirtualBox 等使用虚拟磁盘的工具，自动检测并提示使用官方导出/导入方案迁移，禁止软链接避免损坏。

适用于：Docker、WSL2、虚拟机

常见问题

你可能想了解

关于 C盘小清新的常见疑问解答

迁移缓存后软件还能正常使用吗？
可以。C盘小清新采用 Junction 软链接和环境变量两种安全策略。Junction 方式对软件完全透明，软件读取原始路径时会自动重定向到新位置；环境变量方式则通过官方支持的配置项引导软件使用新路径。迁移完成后重启相关软件即可正常使用。
迁移过程中会不会丢失数据？
不会。迁移操作采用"复制-验证-切换"三步流程：先将缓存数据完整复制到目标位置，验证数据完整性后，再创建软链接或设置环境变量指向新路径。原始数据在确认迁移成功后才会清理。即使中途中断，原始数据也不会丢失。
迁移后不满意可以恢复吗？
可以。C盘小清新提供一键回滚功能，所有已迁移的工具都可以恢复到原始 C 盘路径。回滚过程同样安全可靠，支持批量操作和进度显示。
为什么有些工具标记为"高风险"？
高风险标记意味着该工具的缓存目录包含重要配置或数据（如 IDE 的索引、数据库文件等），迁移时需要关闭相关进程。系统会在迁移前自动检测运行中的进程并提示关闭，确保操作安全。建议按照提示操作即可。
Docker 和 WSL2 为什么不能直接迁移？
Docker Desktop 和 WSL2 使用虚拟磁盘文件（VHDX），这类文件不支持软链接迁移，强制操作会导致数据损坏。C盘小清新会自动识别这类工具并提示使用官方导出/导入方案进行安全迁移。
支持哪些 Windows 系统？
支持 Windows 10 (1903+) 及 Windows 11 的 64 位系统。部分操作（如创建 Junction、修改系统环境变量）需要管理员权限，软件内置权限检测和申请功能。
